## Технологический стек

- **Фреймворк**: React 19 (или последняя 18.x)
- **Сборка**: Vite
- **Язык**: TypeScript (strict mode)
- **Стили**: Tailwind CSS + Headless UI / Radix UI (при необходимости)
- **HTTP**: Axios (рекомендуется)
- **Маршрутизация**: React Router v7 (или TanStack Router)
- **Состояние**: TanStack Query (по умолчанию)

- **Форматирование**: ESLint + Prettier + typescript-eslint
- **Виртуализация**: react-window

## Структура проекта

```bash

├── lib/                  # Утилиты, конфиги, helpers
├── pages/                # Страницы (если не используем file-based routing)
├── routes/               # Конфигурация роутов
├── types/                # Глобальные TypeScript типы
└── utils/                # Чистые функции-утилиты
```

## Правила написания кода

### TypeScript

- Всегда используй **строгую типизацию**.
- Избегай `any`. Используй `unknown` если тип неизвестен.
- Предпочитай интерфейсы (`interface`) для объектов и типов компонентов.
- Используй `type` для union/intersection и utility типов.

### React

- Используй **функциональные компоненты** + Hooks.
- Именование компонентов: **PascalCase**.
- Файлы компонентов: `ComponentName.tsx`
- Каждый компонент должен быть в отдельном файле.
- Избегай ненужных `useEffect` и `useMemo`/`useCallback` без явной необходимости.
- Предпочитай **Server Components** стиль мышления (логика в хуках/сервисах).

### Tailwind CSS

- Используй **только utility-классы** Tailwind.
- Не пиши обычный CSS, если можно решить через Tailwind (кроме редких случаев).
- Следуй порядку классов: **Layout → Box → Typography → Colors → Effects → etc.**
- Используй `@apply` только в редких случаях (в основном в `ui/` компонентах).
- Используй `cn()` утилиту из `lib/utils` для условных классов.

### Именование

- Компоненты: `PascalCase`
- Функции и переменные: `camelCase`
- Константы: `UPPER_SNAKE_CASE` или `PascalCase`
- Файлы: `kebab-case` для страниц и фич, `PascalCase` для компонентов

### Компоненты

- Маленькие, атомарные компоненты в `components/ui/`
- Большие — в `features/` или `components/`
- Используй **compound components** pattern, когда нужно.
- Добавляй `displayName` у компонентов, если используешь devtools.

## Рекомендации по разработке

1. **shadcn/ui** — основной источник UI-компонентов.
2. Используй **Zod** для валидации форм.
3. **TanStack Query** для серверных данных.
4. **Zustand** для клиентского состояния.
